Claude Code 2.1.193 — OpenTelemetry model-response logging, all-shell auto-mode, MCP auth noticesAnthropic · Jun 25–26

The next Claude Code release after 2.1.191. Headline for a regulated shop: OpenTelemetry logging for model responses, with redaction controls — you can now export what the model returned to your observability stack while scrubbing sensitive fields. Also: autoMode.classifyAllShell routes every Bash/PowerShell command through the auto-mode classifier (with denial reasons now recorded in transcripts and permission history), live file-path autocomplete in bash mode, startup notices for MCP servers that need authentication, and automatic memory-pressure reaping of idle background shells.

The OTel model-response logging is the one to note: capturing and redacting model output into your own telemetry is exactly the kind of evidence ASIC/APRA expect for AI used in advice workflows — a record of what the system produced, with PII controls. The all-shell classifier tightens the guardrail when an agent runs commands unattended, and the MCP auth notices stop silent failures where a tool quietly isn't connected. Update if you run Claude Code in pipelines or against client data.

Jalapeño — OpenAI's first custom inference chip, ~50% cheaper inference claimedOpenAI · Broadcom · Jun 25

OpenAI and Broadcom unveiled Jalapeño, a reticle-sized ASIC built specifically for LLM inference (not a repurposed GPU), designed around OpenAI's own model behaviour to cut data-movement and memory bottlenecks. Bloomberg reports it roughly halves inference cost. Built in a ~9-month design cycle using OpenAI's own models; initial deployment is end-2026, scaling toward gigawatt data centres.

Not actionable today — there's no price change on the API now, and you can't run on it. But it's a clear signal on the OpenAI cost curve: if a ~50% inference saving lands in OpenAI's own fleet over 2026–27, expect downward pressure on GPT-tier API pricing (and faster responses). File it under "reasons not to lock a multi-year vendor contract at today's rates," and re-check when it's actually serving traffic with published effects.

DeepSeek — legacy deepseek-chat / deepseek-reasoner aliases retire 24 JulyDeepSeek · deadline Jul 24

If you call DeepSeek's API, the legacy model names deepseek-chat and deepseek-reasoner are fully retired after 24 July 2026, 15:59 UTC — requests using them will fail. They're already transparently routed to deepseek-v4-flash during the grace period. Migrate to deepseek-v4-flash or deepseek-v4-pro; for reasoning, enable thinking mode explicitly via the thinking parameter (the old deepseek-reasoner auto-reasoning is gone).

A one-line code change, but a hard cutover — anything still hardcoding the old names breaks silently after the date. If DeepSeek is anywhere in your stack (including a cheap fallback tier), grep for the legacy strings and pin to a V4 model now, and decide whether you want Flash (cost) or Pro (1M context, stronger reasoning) for each call path.

Your Radar

Two migration deadlines that bite this week, and one operational note.

Gemini CLI stops serving individual tiers on June 18 — migrate to Antigravity CLIGoogle · Jun 18

Per Google's developer blog, on June 18 the Gemini CLI and Gemini Code Assist IDE extensions stop serving requests for Google AI Pro/Ultra, free Gemini Code Assist for individuals, and Gemini Code Assist for GitHub orgs. Enterprise customers on Code Assist Standard/Enterprise are unaffected and keep getting model updates. The successor is the Go-based Antigravity CLI (agy), available now; Agent Skills, Hooks, Subagents, and Extensions carry over (Extensions become Antigravity plugins).

If any pipeline, CI step, or dev machine calls gemini on a Pro/Ultra/free key, it goes dark tomorrow — install agy and re-point your scripts today. Enterprise-licensed setups can stay put. Either way, pin the CLI version in CI so the swap is deliberate, not a surprise failure.

Original Opus 4 & Sonnet 4 snapshots retiredAnthropic · Jun 15

Anthropic's developer platform release notes mark claude-opus-4-20250514 and claude-sonnet-4-20250514 — the original May-2025 snapshots — as retired. Calls to those exact model IDs now fail; the platform directs you to the current 4.x successors (Opus 4.8, Sonnet 4.6).

Grep your codebase and prompts for the -20250514 strings. Anything still hard-pinned to them is now a broken request, not a deprecation warning. Move to claude-opus-4-8 / claude-sonnet-4-6 and keep model IDs in one config constant so the next retirement is a one-line change.

The accountability boundary named — and a Fields Medalist showing what "amplify" actually looks like.

Simon Willison: an LLM agent can never be the "directly responsible individual"simonwillison.net · 12 Jul

Willison's post argues that however capable agents get, they can never hold the DRI role — the Apple-coined term for the one human ultimately accountable for a project's success or failure. His anchor is IBM's 1979 slide: "A computer can never be held accountable, therefore a computer must never make a management decision." The point isn't that agents can't do the work — it's that accountability is a property only a person can carry, so the DRI gap is a permanent design constraint, not scaffolding you remove when the model improves.

This is the whole game for a regulated-advice product. Best-interests duty already assigns a named human as accountable for any AI-generated advice — the exact frame in yesterday's AU governance note. Design so every agent action attaches to a specific human DRI, not a diffuse "the system decided": that's simultaneously a compliance requirement and, per Willison, a boundary that will still hold when the models are far better than today's. Build the attribution in now; you won't be able to retrofit it as a feature later.

The Fields Medalist's post (top of Hacker News on 12 Jul) recounts handing a coding agent ~two dozen mathematical-visualisation applets he'd written in Java since 1999 — long broken by browser changes — and getting working JavaScript ports back in a few hours. In the process the agent surfaced two latent bugs in his original code. A pre-alpha is live at teorth.github.io/tao-web.

The signal isn't "AI writes code" — it's that the economics of legacy maintenance just inverted. Ports that were never worth a human's week are now an afternoon, and the agent doubles as a second reviewer catching bugs the original author missed. For any product sitting on years of accreted calculation logic — tax rules, projection engines, old spreadsheet models — agent-assisted migration plus latent-bug detection is now a realistic maintenance play, not a research demo. Note the shape from the HN thread: one expert steering one agent, not a swarm — human judgement is still the scarce input.

The ETH Zurich context-file study: don't assume your AGENTS.md helps — measure itarXiv · 2602.11988

"Evaluating AGENTS.md: Are Repository-Level Context Files Helpful for Coding Agents?" tested Claude Code (Sonnet), Codex (GPT-5.x) and Qwen Code across SWE-bench Lite (300 tasks) and a 138-repo AgentBench with real developer-written context files, in three variants: none, auto-generated, and human-maintained. Result: auto-generated files reduced success ~0.5pt (SWE-bench) to 2–3pt (AgentBench) while raising inference cost 20–23% — the agent takes more steps and reasons longer. Human-written files added only ~4pt on AgentBench, still at ~20% more cost.

The pattern to adopt is the ablation itself: treat every context file as a hypothesis and A/B it against no-file on your own task suite before you trust it. Most builders have never checked whether their CLAUDE.md is net-positive — this is a one-afternoon eval that could cut a fifth of your agent spend and improve quality. It also reframes context engineering away from "write more guidance" toward "keep only what measurably pays for its tokens" — the discipline you'll want anyway once every agent run is a billed line item.

Claude Code 2.1.187–2.1.191 — ~37% less streaming CPU, memory-leak fix, credential-blocking sandboxAnthropic · Jun 24–25

A run of releases focused on cost and security rather than features. 2.1.191 reduces CPU during streaming by ~37% via text-update coalescing and fixes long-session memory growth from terminal-output caching — both directly relevant if you run Claude Code in long-lived or CI sessions. 2.1.187 added a sandbox.credentials setting that stops sandboxed commands from reading credential files and secret env vars, plus org-configured model restrictions across the model picker, --model, /model and ANTHROPIC_MODEL. Also: MCP retry logic for network errors, fixes for remote-MCP timeouts, rewinding past /clear on resumed sessions, and background agents no longer resurrecting after being stopped.

If you run Claude Code unattended — scheduled pipelines, long agent sessions, shared runners — the CPU and memory fixes lower your compute bill and reduce the OOM risk that kills long jobs. The sandbox.credentials flag is the more important one for a wealthtech stack: it's a concrete, auditable control that keeps an agent from exfiltrating API keys or client secrets, which is exactly the kind of human-bounded guardrail ASIC/APRA expect you to evidence. Org model restrictions let you pin a team to approved models (e.g. block a model that hasn't cleared your data-residency review).

Claude Tag — Anthropic's multiplayer Slack agent on Opus 4.8Anthropic · Jun 23

Anthropic shipped Claude Tag, a beta that replaces the old Claude-in-Slack app with a single shared agent per channel: one @Claude everyone collaborates with, holding its own service accounts, tracking what the team has decided and continuing work after people log off. It connects to your tools, data and codebases and runs on Opus 4.8. Available now for Claude Enterprise and Team plans. Anthropic says 65% of its own product team's code is now written by an internal version of the same agent.

This is a different shape from a per-user assistant: a channel-scoped agent with persistent memory and its own credentials is closer to a teammate you can delegate a standing task to (e.g. "triage support tickets in this channel"). Worth a look if your team already lives in Slack — but two cautions for a regulated shop: an agent holding its own service-account credentials needs the same access review as any privileged identity, and the migration is not optional — workspaces that take no action are auto-migrated on 3 August 2026, so decide before then rather than after.

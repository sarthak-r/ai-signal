Set a goal; a team of agents breaks it into kanban tasks, self-claims the work, and reports in channels you can read — with two gates baked in: an LLM judge verifies every deliverable before a task is allowed to close, and anything risky waits for your approval. Humans and agents are first-class members of the same channels reading the same history. From $29 for a team, or self-host free under MIT with your own model keys.

Two patterns worth stealing regardless of the product. First, the done-gate: a task can't move to done until a separate model certifies the output — "output instead of chatter," structurally enforced, not hoped for. Second, human approval as a first-class state on risky actions — the same gate-the-action instinct from earlier this week, now a workflow primitive. Codacy's Verity: a pre-commit gate that lets a different model block or auto-repair agent code — and remembers every callShow HN · 5 Jul

An "adversarial review" layer for coding agents: before a change is committed, a separate model plus deterministic analysis reviews it against security, quality and intent — code either passes or is blocked with specific findings and auto-repair suggestions. Two things make it more than a linter: every decision writes to an editable Markdown knowledge base committed to your repo, so later sessions start with accumulated context; and it tracks token spend per model, session and run across your agent fleet. Free public beta, works with Claude Code.

Verity productizes Willison's "have a second model mark the homework" as a gate that sits in front of commit — you don't have to remember to run the cross-model review; the pipeline won't let code through without it. The compounding repo-memory is the sharper idea: the gate gets smarter about your codebase over time instead of re-deriving standards each session. For a compliance-sensitive shop, an in-repo, human-editable record of what was checked and why is close to an audit trail for free.

peek-cli: least-privilege browser vision for CLI coding agents — screenshots only, no DOM, no injectionShow HN · 5 Jul

Gives Claude Code, Codex and Copilot the ability to see a rendered browser tab — a Chrome extension streams screenshots over a local WebSocket to the agent. The design choice is the point: the agent can request a screenshot and nothing else — it never touches the DOM, can't inject scripts, and can't take actions. Chrome Web Store-approved, installed as a skill/plugin per agent.

The interesting move is least-privilege perception: most "give the agent a browser" tools hand over full control; peek-cli closes the "the agent can't see what it built" gap while deliberately withholding the ability to do anything. That's a reusable safety pattern — grant an agent the narrowest capability that solves the task (look, don't touch) rather than the broadest. Handy immediately for front-end work where the agent keeps editing CSS blind.

Build Patterns

Two Show HN tools turn "is my agent OK?" from a vibe into a measurement — and the whole front page is now agent self-instrumentation.

contextrot: measure exactly where your Claude Code degrades — from its own session log, with confidence intervalsShow HN · 5 Jul

Instead of guessing when to /compact, contextrot reads your local session transcripts and extracts five independent failure signals per step — edit failures, retry loops, re-reads of already-seen files, self-corrections ("let me fix that"), and tool errors — then correlates each with context fill at that moment. It plots a rot curve with Wilson 95% confidence intervals, controls for the confound that hard steps come later, and returns one of four verdicts (context rot detected / edge rot / no measurable rot / not enough data). Zero network calls — fully local.

This is the empirical answer to the "context rot" everyone's been asserting since briefing 017: it tells you your project's actual degradation threshold, not a rule of thumb. Two concrete uses — set your compaction/hand-off trigger from data rather than superstition, and add the same signals (retry loops, re-reads, self-corrections) as a health metric on your own agent in production. When an agentic feature silently gets worse, this is the shape of the dashboard that catches it.

Aletheia: an agent that tracks its own uncertainty and will answer "INCONCLUSIVE"Show HN · 5 Jul

A verification agent for Claude Code and Codex built as an explicit belief → act → observe → update loop (a POMDP): it forms a belief about what's true, picks the single search most likely to reduce uncertainty, and — the load-bearing rule — contradictions lower its confidence rather than being explained away. It stops only on a dual gate (enough confidence and settled uncertainty), tracks independent unknowns separately to dodge halo reasoning, and will return a calibrated INCONCLUSIVE with its evidence chain when the case isn't earned.

The design principle matters more than the repo: an honest "I don't know" is a feature, and calibration beats a confident guess. For financial-advisory AI this is the whole ballgame — the failure mode that hurts a client and a licensee isn't "the model declined," it's "the model was confidently wrong." Building a stated-confidence, contradictions-count-against, refuses-when-unsure loop into any recommendation surface is directly best-interests-aligned. And note the wider signal: nearly the entire 5 Jul Show HN front page — contextrot, Aletheia, Verity, plus context-bridge and guided-review tools — is agent observability and governance. That's where the app-layer oxygen is right now.

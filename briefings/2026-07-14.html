Ship Log

The tools that ship this week guard the agent, not just drive it.

dcg: a Rust safety hook that blocks an agent's destructive commands before they run — across every major coding agentGitHub · trending 13 Jul

Started as a Python script by Jeffrey Emanuel and ported to Rust by Darin Gordon, destructive_command_guard intercepts catastrophic commands — rm -rf ./src, git reset --hard, DROP TABLE users — before an agent executes them. It runs as a hook in Claude Code, Codex CLI, Copilot CLI, Gemini CLI and Cursor, claims sub-millisecond SIMD-accelerated filtering, ships 50+ modular security packs (databases, Docker, k8s, cloud), and distinguishes real commands from data (a grep result containing "rm -rf" won't trip it).

This is Willison's throughput problem made concrete: the faster agents write and run, the more one bad command can wipe uncommitted work. A deterministic pre-execution gate is cheaper insurance than any prompt-level "please be careful." For a regulated-data product, the same pattern generalises — a policy hook that hard-blocks writes an agent should never make (mass deletes, un-approved sends) is the kind of "compliant by construction" primitive that turns agent adoption from a risk into a controllable one. Note the fail-safe default here is allow on analysis failure; for finance you'd want fail-closed.

A new shape for search, and a small reproducibility trick worth stealing.

JustVibe: a search box that generates a working app instead of returning linksProduct Hunt · 13 Jul

Launched on Product Hunt, JustVibe reframes the query box: type what you want and it generates an application directly rather than a page of blue links. It's a small demo, but it's a clean instance of the interaction pattern the app-builder wars are circling — the answer to a search is a built artifact, not a list of sources to go read.

The pattern to notice, not the product: as generation gets cheap, "search → results → you build it" collapses into "intent → artifact." For any tool with a query surface, ask whether the right response is a generated interface (a filled form, a live projection, a mini-tool) instead of a document. In wealthtech that's the difference between returning a compliance article and returning the pre-filled worksheet — same query, far higher leverage, and a UX moat a link list can't match.

A protocol deadline two weeks out, and a marketing surface you don't yet control.

MCP 2026-07-28 spec RC is out — stateless core, OAuth/OIDC auth, six breaking changes, live in two weeksmodelcontextprotocol.io · this week

The release candidate and beta SDKs for the 2026-07-28 MCP spec — the largest revision since launch — are published. The protocol goes stateless: Mcp-Session-Id, the initialize handshake and sticky routing are gone (client info now travels in _meta on every request); auth aligns to OAuth/OIDC; a missing-resource error changes from -32002 to -32602; and Roots, Sampling and Logging are deprecated. Production servers have until 28 Jul to comply.

If you ship or consume MCP servers, this is a hard, dated migration — two weeks out. The upside is real: a stateless server drops behind a plain round-robin load balancer with no shared session store, and clients can cache tools/list for the server's ttlMs. Test against the beta SDKs now; the OAuth/OIDC alignment also dovetails with the agent-identity/attribution thread the week has been tracking — scoped auth is becoming table-stakes at the protocol layer, not just the app.

If your agent solves the same class of problem twice and learns nothing between runs, the missing primitive is a skill store, not a bigger model. Design for accumulation.

mcpsnoop: "Wireshark for MCP" — watch every request between an agent and its serversShow HN · 3 Jul

A single-binary Go TUI that sits transparently between an AI client (Claude Desktop, Cursor, Claude Code) and its MCP servers, showing every JSON-RPC request/response live: color-coded frames, hung-call detection with timers, filtering by tool/status/direction, capability-handshake inspection, and replay of captured calls against a fresh server. Install via go install, Homebrew, or prebuilt binary.

MCP debugging today is guesswork from logs, and the official Inspector is a separate client, not the real pipe. This shows the actual exchange — exactly what you need when a tool silently isn't called or a request hangs. This is the observability half of the "steer and debug the agent" question: you can't govern a tool loop you can't see. Keep it in the kit alongside Manufact and Spark — the MCP stack now has hosting, distribution, and a debugger.

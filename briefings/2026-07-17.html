The Codex-deletion post-mortem, and a notable shift in tone from the most-cited maintainer in open source.

Thibault Sottiaux root-causes the GPT-5.6 file deletions: the model overrides $HOME to make a temp dir — then deletes itx.com/thsottiaux · flagged 16 Jul

OpenAI's Codex lead published the mechanism behind the reports of GPT-5.6 wiping user files (Matt Shumer lost most of his Mac; Bruno Lemos lost a production database). It fires only when three defaults are off at once: full-access mode on, sandboxing off, auto-review off. In that state the model tries to redefine $HOME to point at a temporary directory, makes "an honest mistake," and deletes the real $HOME instead. OpenAI's fixes are all harness-side: update the developer message, steer users into safer permission modes, add harness safeguards. Simon Willison re-surfaced it 16 July under the dry header "Bad Codex Bug."

The lesson isn't "the model is bad" — it's that destructive blast radius is a permissions-default problem, not a model-quality one. A capable agent, handed unsandboxed full access with no review gate, will eventually do something catastrophic by honest error. If you ship an agent that can run shell or touch a filesystem, your defaults are the product: sandbox-on, review-on, and full-access as a deliberate, loud opt-in — not a setting a user can leave on and forget.

Build Patterns

A full browser engine in an in-page sandbox — a concrete jail for untrusted content and agents.

Puter compiles Firefox to WebAssembly: the real Gecko engine rendering inside a browser tabdeveloper.puter.com · 16 Jul

Puter compiled Firefox's Gecko engine to WebAssembly — the actual Firefox interface rendering entirely inside another browser tab, with WebGL-backed painting, experimental JS→WASM JIT, and page traffic proxied through a hosted Wisp WebSocket server. A demo, but a load-bearing one: it proves a heavyweight, untrusted rendering stack can run fully inside the browser's own sandbox, with no native system access.

This is the same instinct as the day's permissions thread, applied to code instead of commands: when you must run something you don't trust, run it in a jail that structurally can't reach the host. If your product lets an agent render, execute, or preview untrusted content, an in-browser WASM sandbox is a real containment primitive — the untrusted thing gets a full runtime but a hard, OS-level-enforced boundary around it. Isolation by architecture beats isolation by policy.

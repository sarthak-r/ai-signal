Agent SDK usage now bills from a separate, capped poolAnthropic · Jun 15

Effective today, usage via the Claude Agent SDK, claude -p, Claude Code GitHub Actions, and any third-party app built on the Agent SDK no longer draws from your Claude subscription limit. It draws from a separate monthly credit — ~$20 (Pro) / $100 (Max 5x) / $200 (Max 20x) — metered at full API rates, no rollover. Interactive use (web/desktop/mobile chat, Claude Code in the terminal, Cowork) is unchanged.

Any unattended Claude automation — CI agents, scheduled jobs, an Agent-SDK feature inside your product — now meters separately and stops when the credit is exhausted unless you opt into overflow billing (one-time toggle, persists across cycles). The credit isn't on by default: enable it before your next automated run or those calls start failing. Running on a Platform API key? Nothing changes — pay-as-you-go continues, but you get no free monthly credit.

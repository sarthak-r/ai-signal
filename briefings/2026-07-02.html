Willison: give an agent a camera — shot-scraper video lets agents record proof their work runsSimon Willison · 30 Jun

Simon Willison shipped shot-scraper video: an agent writes a YAML "storyboard" (start command, URL, viewport, scenes of clicks/fills/waits), and Playwright drives the browser and encodes a video. The point isn't screenshots — it's that a coding agent can now produce a video demo proving a feature actually works, closing the "it says it's done, but does it run?" gap. Willison's sharper meta-point: the tool's --help is written to be complete enough that an agent learns the tool unaided — "bundling a SKILL.md directly inside the tool."

Two build patterns in one. First: agent self-verification — wire visual proof into the agent's own loop instead of trusting its self-report. Second: docs-as-skill — design your CLIs and internal tools so their help output teaches an agent to drive them, and you stop hand-writing integration glue for every new agent.

MCP's biggest revision since launch ships 28 July — sessions gone, OAuth 2.1 inMCP · deadline

The MCP 2026-07-28 spec (RC out since 21 May) is the largest protocol change since launch. It removes the session model entirely — the initialize handshake and Mcp-Session-Id are gone, so servers run stateless behind plain round-robin load balancers — replaces upfront capability negotiation with a server/discover method, and rebuilds auth on OAuth 2.1 + OpenID Connect (servers become RFC 9728 resource servers; mandatory issuer verification to stop token mix-up). Roots, sampling, and logging are dropped from core.

If you run or consume any MCP server in production, this is a migration, not a version bump — plan for it before 28 July. The upside is real: stateless servers are far easier to scale, and the OAuth 2.1 alignment is what finally makes MCP defensible in a compliance-bound shop. Audit your servers for session-state assumptions now.

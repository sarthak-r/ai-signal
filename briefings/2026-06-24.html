Mistral OCR 4 — structured document extraction, self-hostable, $4 / 1,000 pagesMistral · Jun 23

Mistral's new OCR model returns more than text: every request yields typed blocks (titles, tables, equations, signatures), bounding boxes, per-block confidence scores and markdown-structured output, across 170 languages at ~2,000 pages/minute. It tops OlmOCRBench (85.20) and OmniDocBench (93.07), with annotators preferring it over every tested system at a 72% average win rate. Pricing is $4 / 1,000 pages via API, $2 batch, $5 through Document AI. Available now via Mistral API, Studio, SageMaker and Microsoft Foundry — and critically, it runs in a single container for fully self-hosted, air-gapped deployment.

This is the most directly usable release for a wealthtech stack in weeks: SoAs, super statements, tax docs and ID verification are exactly the structured-extraction problem OCR 4 targets, and confidence scores give you a clean threshold for routing low-certainty pages to human review — a defensible control under ASIC/APRA's "AI is a tool, not a replacement" stance. The self-hosted single-container path means you can process client PII without it leaving your VPC, sidestepping the cross-border data questions that block hosted OCR. At $2–4 per 1,000 pages it undercuts AWS Textract for most document mixes.

Claude Code 2.1.186 — headless MCP login, automatic bash-output responsesAnthropic · Jun 23

New claude mcp login <name> / logout commands authenticate MCP servers straight from the CLI — no interactive /mcp menu — with --no-browser stdin redirect to complete auth over SSH. Behaviour change worth noting: ! bash commands now make Claude respond to their output automatically (set "respondToBashCommands": false to restore the old context-only behaviour). Also: a fix for streaming requests failing with "Content block not found" after the machine wakes from sleep, /workflows status filtering, and Workflow schema-validation subagents now abort after 5 failures instead of looping forever.

The MCP CLI auth flow is the unlock for unattended agents: you can now log a server in inside a CI job or a remote box without a browser handoff, which is what you need to run Claude Code against your own MCP tools (e.g. an internal planning server) in a scheduled pipeline. If you script ! commands in any automated Claude Code flow, audit them before upgrading — the new default turns every shell call into a model turn, which changes both token spend and behaviour.
